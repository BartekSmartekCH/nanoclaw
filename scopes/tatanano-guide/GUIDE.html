1. Introduction — What is TataNano?

TataNano is your personal AI assistant powered by NanoClaw — an open-source platform that connects Claude (Anthropic's AI) to messaging apps like Telegram. It runs on your Mac Mini inside Docker, and you interact with it entirely through Telegram chat.

Think of it this way:

  • Telegram is how you talk to TataNano (the “front door”)
  • NanoClaw is the engine that processes your messages and runs Claude
  • Skills are plug-in capabilities that teach TataNano new tricks
💡
Key insight: TataNano can modify its own files and install new skills when you ask it to. You don't need terminal access or coding knowledge — you can extend it through conversation.

What can TataNano do out of the box?

  • Answer questions and have conversations
  • Search the web and fetch content from URLs
  • Browse websites with a built-in browser (click, fill forms, take screenshots)
  • Read and write files in its workspace
  • Run bash commands in its sandboxed environment
  • Schedule tasks (reminders, recurring checks)
  • Remember things across conversations

What can it learn to do?

Anything you can describe! By adding skills, TataNano can learn to send files, generate reports, interact with APIs, automate workflows, and much more. This guide walks you through the process with a real example.

The flow, step by step

  1. You send a message in Telegram (e.g., @TataNano send me a PDF of today's report)
  2. The NanoClaw host process (Node.js on your Mac Mini) receives the message via the Telegram Bot API using the grammy library
  3. It spins up a Docker container — a safe, isolated Linux environment where Claude runs
  4. Inside the container, Claude has access to tools (file I/O, web search, bash), skills (SKILL.md instructions), and the MCP server (NanoClaw's own tools like send_message)
  5. When Claude wants to send you a reply, it calls the MCP send_message tool, which writes a JSON file to the IPC directory
  6. The host's IPC watcher picks up the JSON file (polling every 500ms) and sends the text through the Telegram Bot API back to your chat
🔒
Security note: Each conversation runs in an isolated container. TataNano cannot access your Mac Mini's files unless you explicitly mount directories. The container runs as a non-root user (node), and the NanoClaw source code is mounted read-only.

Understanding IPC — the messaging backbone

IPC (Inter-Process Communication) is how the container talks back to the host. The pattern is simple:

  1. Claude calls an MCP tool (e.g., send_message)
  2. The MCP server writes a JSON file to /workspace/ipc/messages/
  3. The host's IPC watcher picks it up within 500ms
  4. The host routes it to the right channel (Telegram, WhatsApp, etc.)

This is important to understand because when we add new capabilities (like sending files), they follow this same IPC pattern.

3. How Skills Work

A skill is simply a text file (SKILL.md) placed in the right directory. It tells Claude how to perform a specific task, what tools it is allowed to use, and what steps to follow. Think of it like a recipe card — Claude reads the instructions and follows them.

✅
No compilation needed! Skills are just markdown files. No build step, no restart, no complex configuration. Write the file, and Claude discovers it automatically at the start of the next session.

Anatomy of a skill file

Each skill is a folder inside /home/node/.claude/skills/ containing a SKILL.md file:

/home/node/.claude/skills/
  └─ my-skill-name/
     └─ SKILL.md ← The skill definition

A SKILL.md file has two parts:

Part 1: Frontmatter (metadata)

A YAML block at the top between --- markers:

YAML---
name: my-skill-name
description: A short description of what this skill does.
  Claude uses this to decide when to activate the skill.
allowed-tools: Bash(my-tool:*)  # optional - extra tool permissions
---
FieldRequired?Purpose
nameYesUnique identifier (lowercase, hyphens). Also the /command name.
descriptionYesTells Claude when to use this skill. Be specific about trigger conditions.
allowed-toolsNoGrants extra tool permissions (e.g., specific bash commands).

Part 2: Instructions (body)

Markdown text that tells Claude exactly what to do. This can include step-by-step instructions, code templates, bash commands, examples, and rules. The more specific you are, the better Claude will follow the instructions.

How Claude discovers and uses skills

When Claude starts a session, it scans /home/node/.claude/skills/ for skill folders. Each skill's description field helps Claude decide when to use it:

  • If you say “browse this website,” Claude matches that to the agent-browser skill
  • If you say “what can you do,” Claude matches that to the capabilities skill
  • You can also invoke a skill directly by typing its name as a slash command: /my-skill-name

4. Worked Example: Adding a “Send PDF” Skill

Let's walk through a complete, real example: adding a skill that lets TataNano send PDF files directly in your Telegram chat. We'll cover two approaches — pick the one that suits your comfort level.

What we're building

A skill that allows TataNano to:

  • Generate a PDF from any content (web page, HTML, text)
  • Send it as a document attachment in your Telegram chat
  • Respond to requests like “Send me a PDF of this article” or “Create a PDF report”

Choose your approach

✅
Recommended for Bartek. This approach requires no code changes to NanoClaw — just creating a single SKILL.md file. TataNano uses its built-in agent-browser tool to generate PDFs and curl to send them via the Telegram Bot API.

The complete SKILL.md

SKILL.md---
name: send-pdf
description: Send PDF files or documents via Telegram chat. Use when the user
  asks to send a PDF, create a PDF document, export something as PDF, share a
  file as a document, or generate a printable version of content.
allowed-tools: Bash(agent-browser:*) Bash(curl:*)
---

# Send PDF via Telegram

Generate PDFs and send them as document attachments in the Telegram chat.

## Step 1: Generate the PDF

Use `agent-browser` to create a PDF. Two options:

### From a URL:
```bash
agent-browser open "https://example.com/page"
agent-browser wait --load networkidle
agent-browser pdf /tmp/output.pdf
agent-browser close
```

### From custom HTML content:
Write the content to a temporary HTML file, then render it:
```bash
cat <<'HTMLEOF' > /tmp/content.html
<!DOCTYPE html>
<html>
<head><style>
  body { font-family: -apple-system, sans-serif; padding: 2rem; max-width: 800px; margin: 0 auto; }
  h1 { color: #1a1a1a; } p { line-height: 1.6; }
</style></head>
<body>
  <h1>Report Title</h1>
  <p>Your content here...</p>
</body>
</html>
HTMLEOF

agent-browser open "file:///tmp/content.html"
agent-browser pdf /tmp/output.pdf
agent-browser close
```

## Step 2: Send via Telegram Bot API

Use curl to send the PDF as a document using the Telegram sendDocument API.

The bot token is in the NanoClaw environment. The chat ID is available from
the NANOCLAW_CHAT_JID environment variable (format: "tg:-1001234567890").

```bash
# Get the numeric chat ID (strip the "tg:" prefix)
CHAT_ID=$(echo "$NANOCLAW_CHAT_JID" | sed 's/^tg://')

# Send the document
curl -s -X POST "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendDocument" \
  -F "chat_id=${CHAT_ID}" \
  -F "document=@/tmp/output.pdf" \
  -F "caption=Here is your PDF document" \
  -F "parse_mode=Markdown"
```

## Step 3: Clean up

```bash
rm -f /tmp/output.pdf /tmp/content.html
```

## Important Rules

- Always wait for the page to fully load before generating the PDF
  (`agent-browser wait --load networkidle`)
- Telegram has a **50 MB file size limit** for documents
- If the bot token is not available in the environment, inform the user:
  "I need the Telegram bot token to send files. Please ask your admin
  to set TELEGRAM_BOT_TOKEN in the NanoClaw environment."
- Always clean up temporary files after sending
- For existing files in the workspace, skip generation and go straight
  to the curl command with the file path
- Use a descriptive caption that tells the user what the PDF contains

## Sending an Existing File

If the user already has a file (PDF, DOCX, ZIP, etc.) in the workspace:

```bash
CHAT_ID=$(echo "$NANOCLAW_CHAT_JID" | sed 's/^tg://')
curl -s -X POST "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendDocument" \
  -F "chat_id=${CHAT_ID}" \
  -F "document=@/workspace/group/path/to/file.pdf" \
  -F "caption=Your requested file"
```

For advanced users or future NanoClaw development. This approach adds a proper send_document MCP tool that works through the IPC system, just like the built-in send_message tool. It requires modifying three files in the NanoClaw source code.

The proper way to add file sending follows the same IPC pattern that send_message uses. Three files need changes:

Change 1: Add MCP tool in the container

File: /app/src/ipc-mcp-stdio.ts

Add a new send_document tool alongside the existing send_message:

TypeScript// Add after the existing send_message tool definition

server.tool(
  'send_document',
  'Send a file (PDF, image, etc.) as a Telegram document attachment.',
  {
    file_path: z.string().describe('Absolute path to the file inside the container'),
    caption: z.string().optional().describe('Caption text for the document'),
  },
  async (args) => {
    const data = {
      type: 'document',
      chatJid: chatJid,
      filePath: args.file_path,
      caption: args.caption || undefined,
      groupFolder: groupFolder,
      timestamp: new Date().toISOString(),
    };

    writeIpcFile(path.join(IPC_DIR, 'files'), data);

    return { content: [{ type: 'text', text: 'Document queued for sending.' }] };
  },
);

Change 2: Handle file IPC in the host watcher

File: src/ipc.ts (on host)

Add a files directory handler in processIpcFiles(), alongside the existing messages and tasks handlers:

TypeScript// Add inside the for loop, after the messages handler

const filesDir = path.join(ipcBaseDir, sourceGroup, 'files');
if (fs.existsSync(filesDir)) {
  const fileEntries = fs.readdirSync(filesDir)
    .filter(f => f.endsWith('.json'));
  for (const file of fileEntries) {
    const filePath = path.join(filesDir, file);
    const data = JSON.parse(fs.readFileSync(filePath, 'utf-8'));
    if (data.type === 'document' && data.chatJid && data.filePath) {
      await deps.sendDocument(data.chatJid, data.filePath, data.caption);
    }
    fs.unlinkSync(filePath);
  }
}

Change 3: Add sendDocument to the Telegram channel

File: src/channels/telegram.ts (on host)

Add a sendDocument method to the TelegramChannel class, next to the existing sendMessage method:

TypeScriptasync sendDocument(jid: string, filePath: string, caption?: string): Promise<void> {
  if (!this.bot) {
    logger.warn('Telegram bot not initialized');
    return;
  }
  try {
    const numericId = jid.replace(/^tg:/, '');
    const fileBuffer = fs.readFileSync(filePath);
    const fileName = path.basename(filePath);
    const inputFile = new InputFile(fileBuffer, fileName);

    await this.bot.api.sendDocument(numericId, inputFile, {
      caption: caption,
      parse_mode: 'Markdown',
    });
    logger.info({ jid, filePath, fileName }, 'Telegram document sent');
  } catch (err) {
    logger.error({ jid, filePath, err }, 'Failed to send Telegram document');
  }
}

6. Tips & Best Practices

Writing good skills

  • Be specific in the description — include trigger phrases like “Use when the user asks to...”. Claude relies on this to decide when to activate the skill.
  • Include concrete examples — showing Claude a complete bash command or workflow is more effective than describing it abstractly.
  • One skill, one job — a focused skill is more reliable than one that tries to do everything.
  • Add error handling — tell Claude what to say when things go wrong (e.g., “If the file doesn't exist, tell the user.”).
  • Start simple, iterate — get a basic version working first, then ask TataNano to improve it.

7. Troubleshooting

❌
“Skill not found” or TataNano ignores the skill
  • Verify the file exists: ask “Read /home/node/.claude/skills/skill-name/SKILL.md”
  • Check the folder structure: must be skills/skill-name/SKILL.md (not skills/SKILL.md)
  • Ensure the description field matches your request type
  • Try invoking directly: /skill-name
  • Skills are discovered at session start — a new message (new session) may be needed
❌
Skill worked before but stopped working
  • Check if the file still exists (it may have been lost during a container rebuild)
  • Restore from backup: “Copy /workspace/group/skill-backups/send-pdf/SKILL.md to /home/node/.claude/skills/send-pdf/SKILL.md”
❌
TataNano is not responding at all
  • Check that Docker Desktop is running on your Mac Mini
  • Check the NanoClaw container is running (green status in Docker Desktop)
  • Restart: in Docker Desktop, click the container and hit “Restart”
  • Check logs: Docker Desktop → container → “Logs” tab
❌
Bot token issues
  • Ask TataNano: “Run: echo $TELEGRAM_BOT_TOKEN | head -c 10” (shows first 10 chars to confirm it's set without exposing the full token)
  • Verify the token is valid: curl https://api.telegram.org/bot{TOKEN}/getMe
  • If missing, the token needs to be added to the NanoClaw environment configuration on your Mac Mini
