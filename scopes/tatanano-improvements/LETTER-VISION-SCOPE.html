📬 Letter Skill & Vision Infrastructure

NanoClaw Improvements — Scope Document
Authored: March 20, 2026  ·  Status: Proposed

Overview

This scope covers two layers of work: vision infrastructure (enabling NanoClaw agents to see images sent via Telegram) and the /letter skill (a conversational assistant for understanding and replying to foreign-language official letters).

The vision infrastructure is foundational — once built, any future skill (receipts, forms, ID documents) benefits from it automatically. The letter skill is the first consumer of that infrastructure.

Core motivation: Bartek receives official letters in German, French, and Italian. He wants to photograph them, send to Telegram, and get a plain-English summary + a draft reply — processed privately on the Mac mini, with Claude as an optional step only.

Architecture: Three Layers

Layer What it does Type
1. Telegram Image Vision Downloads photos & PDFs from Telegram and passes them to Claude as multimodal content blocks Infrastructure
2. Local OCR + AI Stack Apple Vision (text extraction) + Ollama Qwen2.5-VL (document understanding) — 100% on-device, private Infrastructure
3. /letter Skill Orchestrates the pipeline: OCR → summarise → draft reply → optional PDF output Skill

Layer 1: Telegram Image Vision Infrastructure

Currently, photos sent to Telegram are stored as the placeholder [Photo] — the agent never sees the actual image. This layer fixes that.

What changes

One file: src/channels/telegram.ts. The current handler:

this.bot.on('message:photo', (ctx) => storeNonText(ctx, '[Photo]'));

Is replaced with a handler that:

  1. Selects the largest available photo size from the Telegram update
  2. Calls ctx.getFile() (grammY) to get the download URL
  3. Downloads the image via the Telegram Bot API file endpoint
  4. Resizes to max 1024px using sharp (reduces token cost)
  5. Encodes as base64 JPEG and passes to the agent as a multimodal content block
  6. Any caption text is included as the text part alongside the image

Document and PDF handling

A parallel handler for message:document covers:

  • Images sent as files (Telegram uncompressed): same flow as photos
  • PDFs: Claude accepts PDFs natively as application/pdf base64 content blocks (up to 100 pages) — no conversion needed
  • MIME type detection routes each document to the correct handler

Dependencies

  • sharp — for image resizing. Likely already installed (WhatsApp vision uses it). If not: npm install sharp, native Apple Silicon binaries available.

Layer 2: Local OCR + AI Stack Infrastructure

Privacy-first processing: letters are understood entirely on the Mac mini. No content leaves the home network unless Claude is explicitly invoked.

Apple Vision Framework (OCR)

macOS's built-in Live Text engine — the same technology used when you point your iPhone camera at text. Available via an npm bridge (mac-system-ocr) from Node.js.

PropertyValue
Quality (printed European text)9/10 — excellent
Speed<5 seconds per page
LanguagesGerman, French, Italian, Polish + 12 others — native
Photo supportYes — phone photos with shadows, angles, varying lighting
HandwritingHit or miss — not reliable
Setupnpm install mac-system-ocr — 30 minutes
CostZero — runs on macOS, no model downloads
Apple Vision extracts raw text only. It does not understand the document's meaning — that is Ollama's job.

Ollama + Qwen2.5-VL (Document Understanding)

A small vision-language model running locally via Ollama. Reads the image directly and understands it like Claude would — identifying the sender, what they want, deadlines, tone — but entirely on-device.

PropertyValue
ModelQwen2.5-VL 7B (4-bit quantised)
Quality (document OCR + understanding)8/10
Speed on Apple Silicon30–60 seconds per document
RAM required8–12 GB (4-bit quantisation)
Disk~4.5 GB
Ollama statusNot yet installed — needs setup
SetupInstall Ollama → ollama pull qwen2.5-vl:7b — 15 minutes + download
Always on?Ollama runs as background service; model loads on demand, unloads after timeout
CostZero — fully local, no API calls

Ollama setup steps (one-time)

  1. Download Ollama from ollama.com — macOS app, one click
  2. Run: ollama pull qwen2.5-vl:7b
  3. NanoClaw skill calls Ollama API at http://localhost:11434

Layer 3: /letter Skill Skill

The user-facing skill that orchestrates the pipeline from photo to draft reply.

Trigger

  • Primary: send photo(s) or PDF, then type /letter
  • Shorthand: /letter reply — decline politely to combine in one step
  • Multi-page: send all pages, then /letter — processed as one document

Processing pipeline

📱 Phone photo or PDF → Telegram
→ Apple Vision: text extraction (local, <5s)
→ Ollama Qwen2.5-VL: document understanding (local, 30-60s)
→ Summary delivered to Telegram
→ [If reply needed] Conversational clarification (1–3 questions)
→ Ollama: draft reply (local) OR Claude: draft reply (optional)
→ Reply in original language + English translation
→ [Optional] PDF output via agent-browser

Summary format

Every letter produces a structured summary block:

  • From: sender name and organisation
  • Date: letter date
  • Reference: case/reference numbers
  • What they want: 1–2 sentence plain English explanation
  • Deadline: date or "None mentioned"
  • Key details: bullet points — amounts, conditions, options
  • Tone: Routine / Formal / Urgent / Legal threat
  • Recommended action: Reply needed / Info only / Urgent — act soon / Junk mail

Reply drafting — conversational, not a form

After the summary, the skill asks 1–3 targeted questions based on the specific letter:

  • "They're asking you to choose between Plan A and Plan B. Which?"
  • "They claim you owe CHF 240. Do you want to pay, dispute, or ask for details?"
  • "Is your address still [stored address]?"

For simple letters (e.g., confirming receipt): draft immediately without questions — "Here's a standard confirmation. Look good?"

Draft is in the original language with an English translation below for verification. Bartek can iterate: "make it firmer", "add that I already paid on Feb 12".

Edge cases

ScenarioBehaviour
Junk mail / advertisingFlagged immediately — "This is a promo from [X]. No action needed."
Legal threat / debt collectionURGENT flag, deadline prominent, caution about professional advice
Informational only"No reply needed." Option to note anything from it.
Poor photo qualityIdentifies which part is unreadable, asks to retake
Multi-pageAll pages assembled before processing
Handwritten lettersRoutes to Claude vision (better quality for handwriting)

Stored preferences (letter-profile.md)

Created once, reused for all future letters:

  • Full legal name
  • Current address
  • Preferred reply tone (formal but not stiff)
  • Signature block (name, address, phone, email)
  • Common reference numbers (insurance policy, tax ID — added organically)

Collected organically: "I'll need your address for the reply. What should I use?" — stored, never asked again.

Recommended Build Order

  1. Telegram Image Vision — unlocks all image features for all groups (main, tutor, future skills). Highest leverage change.
  2. /letter skill with Claude vision — works immediately after ①, no local stack needed. Ship fast, validate the UX.
  3. Local OCR layer — install Ollama, add Apple Vision, switch the pipeline to local-first with Claude as fallback. Privacy upgrade.
Note: Step 2 can be skipped and done later — the skill works fine with Claude vision. Step 3 is the privacy upgrade that makes the letter skill fully self-contained on the Mac mini.
