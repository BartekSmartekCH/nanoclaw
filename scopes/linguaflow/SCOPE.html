App Concept & Vision

LinguaFlow is a personalized, adaptive language learning application that combines proven pedagogical methodologies with modern AI-driven personalization. Unlike gamified flashcard apps, LinguaFlow is designed to feel like a personal tutor — adapting to each learner's native language, proficiency level, goals, and error patterns to deliver genuinely effective language instruction.

Core Principle: The app should feel like a tutor, not a test. Feedback is encouraging, specific, and educational — even when correcting errors.

Founding Pedagogical Pillars

  • Comprehensible Input (Krashen's i+1) — Foundation for all content delivery
  • Spaced Repetition (FSRS) — For long-term retention
  • Task-Based Language Teaching (TBLT) — For real-world skill application
  • Scaffolded Production — Recognition → guided production → free production
  • Native Language (L1) Awareness — Core differentiator

Key Features

Adaptive Learning Engine

  • Bayesian Knowledge Tracing (BKT): Every concept is modeled as a node in a learner knowledge graph with a mastery probability, updated after every interaction.
  • FSRS Spaced Repetition Scheduler: Reviews items in context (example sentences, mini-dialogues) rather than isolated flashcards. More accurate than SM-2.
  • i+1 Content Selection: The engine selects content that is ~90–95% comprehensible to the learner, with just enough new material to stretch them.
  • Error Pattern Detection: Distinguishes between mistakes (slips) and errors (systematic knowledge gaps). Only errors trigger targeted remediation.

L1 Interference-Aware Personalization

  • An L1 interference matrix per language pair identifies predictable challenges based on the learner's native language.
  • Example: Mandarin speakers learning English receive extra practice on articles (a/the); Spanish speakers learning French get focused work on false cognates.
  • Content is tagged with L1 interference patterns, enabling the engine to surface targeted exercises.

Present–Practice–Produce (PPP) Learning Loop

  1. Present: New material via comprehensible input (dialogues, short texts, audio)
  2. Practice: Guided exercises — recognition → guided production (cloze, reordering, word bank)
  3. Produce: Freer production tasks (typing, structured speaking, task-based challenges)
  4. Review: FSRS-scheduled review of previously learned material, always in context

Thematic + Spiral Curriculum

  • Content organized into thematic units (e.g., "Food & Dining", "At the Airport") for real-world relevance.
  • Spiral curriculum (Bruner): Grammar and vocabulary recur across themes at increasing complexity.
  • Each unit contains 5–8 lessons of ~10–15 minutes, mixing vocabulary, grammar, and skills.

Computer-Adaptive Placement Testing (CAT)

  • Uses Item Response Theory (IRT) for question calibration.
  • 15–20 adaptive questions covering vocabulary, grammar recognition, and reading comprehension.
  • Places learners across CEFR levels (A1–C2), supplemented by self-report (native language, goals).

Skill Balance System

  • Default: 30% reading, 25% listening, 25% speaking, 20% writing
  • Adjustable by learner goals (conversation, travel, academic, business)
  • Receptive skills weighted at A1–A2; productive skills increase at B1+

Other Features

  • Error-Tolerant Input: Fuzzy matching for typos; constructive, educational feedback tone
  • Cultural Micro-Lessons: 2–3 sentence cultural notes embedded in relevant lessons
  • Gamification: Streaks, XP system, CEFR-tied level progression
  • Adaptive Daily Micro-Learning: FSRS-driven push notifications for between-session retention

Tech Stack Recommendations

Frontend

  • React Native + Expo (iOS & Android)
  • React (web) sharing core component logic
  • TypeScript throughout

Backend

  • Node.js (Express/Fastify) — API layer
  • Python microservice — BKT, FSRS, knowledge graph algorithms
  • PostgreSQL — Primary database (users, progress, content metadata)
  • Redis — Session caching, streak tracking, real-time learner state

AI / ML Services

  • FSRS (open-source) — Spaced repetition scheduling
  • Whisper API (OpenAI) — Speech-to-text for speaking exercises
  • LLM API (Claude or GPT) — Writing feedback & AI conversation partner (Phase 2)

Infrastructure

  • Docker + container orchestration
  • CI/CD pipeline (GitHub Actions)
  • CDN for media delivery
  • Analytics pipeline for anonymized learning data

User Experience Flow

Onboarding (First Session)

  1. Welcome & Language Selection
  2. Goal Setting (conversation, travel, business, academic, general)
  3. Session Preference (5, 15, or 30 minutes/day)
  4. Adaptive Placement Test (15–20 CAT questions)
  5. Personalized Learning Path displayed with CEFR level & first lesson preview

Core Learning Session

  1. Dashboard — today's lesson, SRS queue, streak, XP
  2. Lesson Flow — input → vocabulary → grammar → production → task challenge → cultural note
  3. SRS Review — mixed review always in context
  4. Session Summary — XP earned, accuracy, streak update

Summary

LinguaFlow differentiates itself through three core principles:

  1. Pedagogically grounded — Every feature is rooted in established language teaching methodology (Krashen, TBLT, PPP, spiral curriculum) rather than pure gamification.
  2. L1-aware personalization — Native language interference patterns drive content selection, making practice targeted and efficient.
  3. Tutor, not test — Feedback is encouraging, explanatory, and constructive. The app teaches even when correcting.
The MVP (Phase 1) delivers genuine learning value from day one with a complete adaptive learning loop — not just flashcards with points.
